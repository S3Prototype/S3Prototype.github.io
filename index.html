Buddy Viewer
  • Source code on github: https://github.com/S3Prototype/BuddyViewer/
    •
  • Live Project Link: https://buddyviewer.com
    •

Buddy Viewer is a live full stack webapp for watching synchronized videos from (almost) any source. Users create or join a room, and when any single user presses play, pauses, skips or starts a new video, the same event happens for all other users.

Technical Details:
  • Front-End:
    • Vanilla JS and jQuery on the front end.
    • Uses custom API for sending and receiving video data to and from the server.
    • SocketIO for real-time communication
    • AJAX for search queries.
  • Back-End
    • Custom API designed for sending and receiving video information to and from the client.
    • NodeJS for server runtime. Perfect for asynchronous inputs from users pressing play, pause and seek buttons on video players.
    • Express for routing and API queries.
    • MongoDB with mongoose for serializing data to a database.
    • Redis cache to memoize recent activity for faster response times.

Lessons Learned:
  • Value every member of the team. Working on the backend server logic, database schemas, frontend JavaScript logic, and UX/UI design in CSS+HTML taught me just how challenging every part of a web app is, and just how important every member of a team is.
  • Always research available solutions before creating a plan. Because BuddyViewer requires back and forth communication between users in a room, I initially implemented a crude solution with AJAX. Luckily I eventually discovered web sockets and updated all of my code.
  • Value what is right over what is comfortable. Once I realized I had to replace my AJAX code with web sockets, I hesitated because I worried about losing all of my work. But I converted to web sockets anyway, because it's always better to use the right tool--not the easy tool.
